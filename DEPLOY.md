# Деплой на Timeweb Cloud Apps

> Сценарий: **Timeweb Cloud Apps** (PaaS) + домен на **REG.RU**

---

## Шаг 1. Регистрация в Timeweb Cloud

1. Перейди на [timeweb.cloud](https://timeweb.cloud)
2. Зарегистрируйся или войди
3. Пополни баланс (минимум ~200–400 ₽/мес для статического сайта)

---

## Шаг 2. Создание проекта Apps

1. В панели Timeweb нажми **«Создать» → «Apps»**
2. Выбери **«Импорт из Git»**
3. Нажми **«Подключить GitHub»** и авторизуй Timeweb
4. Выбери репозиторий, который создал Lovable (или создай новый и залей код)
5. Нажми **«Подключить»**

---

## Шаг 3. Настройка сборки

В форме настройки укажи:

| Параметр | Значение |
|----------|----------|
| **Фреймворк / Runtime** | `Node.js` |
| **Команда сборки** | `npm install && npm run build` |
| **Директория сборки** | `dist` |
| **Версия Node.js** | `20` |

> Важно: директория сборки обязательно `dist` — именно туда Vite выгружает готовый сайт.

Нажми **«Создать»**. Timeweb начнёт сборку.

---

## Шаг 4. Проверка временного URL

Через 1–3 минуты Timeweb выдаст временный адрес вида:
```
https://a1b2c3d4.timeweb.cloud
```

Открой его в браузере. Сайт должен открываться корректно.

---

## Шаг 5. Привязка своего домена (REG.RU)

Твой домен `sergeichernenko.ru` зарегистрирован на **REG.RU**. DNS-записи нужно добавлять там.

### 5.1. Получи данные от Timeweb

1. В панели Timeweb перейди в раздел **«Домены»**
2. Нажми **«Добавить домен»**
3. Введи: `aikod.sergeichernenko.ru`
4. Timeweb покажет тебе **IP-адрес** (или CNAME-запись), который нужно прописать

Скопируй это значение.

### 5.2. Добавь DNS-запись в REG.RU

1. Войди в [reg.ru](https://reg.ru) → раздел **«Мои домены»**
2. Найди домен `sergeichernenko.ru` → нажми **«Управление»**
3. Перейди в **«Управление DNS»** или **«DNS-серверы и DNS-записи»**
4. Добавь новую запись:

   - **Тип:** `A`
   - **Имя (Subdomain):** `aikod`
   - **Значение:** `IP_АДРЕС_ИЗ_TIMEWEB`
   - **TTL:** 300 (или 3600)

5. Сохрани запись

> Если Timeweb выдал CNAME вместо IP:
> - **Тип:** `CNAME`
> - **Имя:** `aikod`
> - **Значение:** `имя_хоста.timeweb.cloud` (без `https://`)

### 5.3. Дождись обновления DNS

Обновление DNS занимает от **5 минут до 2 часов**. Обычно 10–30 минут.

Проверить можно командой:
```bash
nslookup aikod.sergeichernenko.ru
```
Должен отобразиться IP Timeweb.

### 5.4. Заверши привязку в Timeweb

1. Вернись в панель Timeweb → **«Домены»**
2. Нажми **«Проверить»** рядом с доменом
3. После успешной проверки нажми **«Привязать»**

---

## Шаг 6. SSL (HTTPS)

Timeweb Cloud Apps **автоматически** выдаёт SSL-сертификат Let's Encrypt для привязанного домена.

- Убедись, что в настройках домена галочка **«HTTPS»** включена
- Подожди 5–15 минут после привязки домена
- Открой `https://aikod.sergeichernenko.ru` — должен открываться без предупреждений

---

## Шаг 7. Автодеплой

В настройках Timeweb Apps включи **«Автодеплой при push в main»**.

Теперь каждый раз, когда ты пушишь изменения в `main` ветку на GitHub:
1. Timeweb автоматически пересобирает проект
2. Деплоит новую версию
3. Сайт обновляется за 1–3 минуты

---

## Шаг 8. Отключение от Lovable (опционально)

Когда сайт стабильно работает на Timeweb:

1. В панели Lovable отключи публикацию (или оставь для разработки)
2. Убедись, что `aikod.sergeichernenko.ru` открывает Timeweb-версию, а не Lovable

## Проблемы и решения

### «Сайт открывается, но при обновлении страницы — 404»

В настройках Timeweb Apps найди опцию **«SPA / Одностраничное приложение»** или **«Fallback to index.html»** и включи её. Это стандартная проблема React Router.

### «DNS не обновляется»

- Проверь, что запись точно сохранилась в REG.RU
- Подожди дольше — иногда до 2 часов
- Проверь через `nslookup aikod.sergeichernenko.ru` — должен показывать IP Timeweb
- Если не помогает — убедись, что DNS-серверы домена указывают на REG.RU (в настройках домена должно быть `ns1.reg.ru`, `ns2.reg.ru`)

### «SSL не выдаётся»

- Убедись, что домен полностью привязан и DNS работает
- Подожди 15–30 минут
- В крайнем случае — напиши в поддержку Timeweb (чат в правом нижнем углу панели)
