# Деплой на Timeweb Cloud — Полное руководство

> Проект: **AI Kod — Мастер-класс по ИИ**  
> Технологии: Vite + React + TypeScript + Tailwind CSS  
> Тип: Статический SPA (Single Page Application)  
> Домен: `aikod.sergeichernenko.ru`  
> GitHub: репозиторий уже подключен к Lovable

---

## Вариант 1: Timeweb Cloud Apps (рекомендуется, проще)

Это PaaS-решение — не нужно настраивать сервер, nginx, SSL. Timeweb сам соберёт и развернёт приложение.

### Шаг 1. Регистрация и вход

1. Перейди на [timeweb.cloud](https://timeweb.cloud)
2. Зарегистрируйся или войди в аккаунт
3. Пополни баланс (минимум ~200 ₽/мес для статического сайта)

### Шаг 2. Создание проекта Apps

1. В панели управления нажми **«Создать» → «Apps»** (или найди раздел Apps в меню)
2. Выбери **«Импорт из Git»**
3. Нажми **«Подключить GitHub»** и авторизуй Timeweb
4. Выбери репозиторий проекта (тот, что создал Lovable)
5. Нажми **«Подключить»**

### Шаг 3. Настройка сборки

В форме настройки укажи:

| Параметр | Значение |
|----------|----------|
| **Фреймворк / Runtime** | `Node.js` (или `Static` / `Vite` если есть в списке) |
| **Команда сборки** | `npm install && npm run build` |
| **Директория сборки** | `dist` |
| **Версия Node.js** | `20` (или `18` и выше) |

> Важно: убедись, что директория сборки именно `dist` — это папка, которую генерирует Vite.

### Шаг 4. Переменные окружения

Этот проект **не требует** переменных окружения (нет API-ключей, нет бэкенда).  
Если в будущем добавишь — нажми **«Добавить переменную»** и укажи ключ=значение.

### Шаг 5. Деплой

1. Нажми **«Создать»** или **«Deploy»**
2. Дождись окончания сборки (обычно 1–3 минуты)
3. Timeweb выдаст временный URL типа `https://a1b2c3d4.timeweb.cloud`
4. Открой его — сайт должен работать

### Шаг 6. Привязка своего домена

1. Перейди в раздел **«Домены»** Timeweb Cloud
2. Нажми **«Добавить домен»**
3. Введи: `aikod.sergeichernenko.ru`
4. Скопируй предложенные DNS-записи (обычно CNAME или A-запись)
5. Перейди в панель управления доменом (где куплен `sergeichernenko.ru`) — скорее всего это Timeweb или REG.RU
6. Добавь запись:
   - Тип: `A`
   - Имя: `aikod` (или `aikod.sergeichernenko.ru`)
   - Значение: IP-адрес из Timeweb Apps
   - Или `CNAME` → имя хоста из Timeweb
7. Дождись обновления DNS (5–30 минут)
8. В Timeweb Cloud нажми **«Проверить»** и **«Привязать»**

### Шаг 7. SSL (HTTPS)

Timeweb Cloud Apps автоматически выдаёт SSL-сертификат Let's Encrypt.  
Просто убедись, что в настройках домена галочка **«HTTPS»** включена.

### Шаг 8. Автодеплой (опционально)

В настройках Apps включи **«Автодеплой при push в main»** — тогда каждый commit в main будет автоматически пересобираться и деплоиться.

## Вариант 2: VDS + Nginx (дешевле, но нужна настройка)

Если хочешь платить меньше (~150–300 ₽/мес) и готов потратить 20 минут на настройку сервера.

### Шаг 1. Заказ VDS

1. [timeweb.cloud](https://timeweb.cloud) → **«Создать» → «VDS/VPS»**
2. Выбери минимальную конфигурацию:
   - **ОС:** Ubuntu 22.04 LTS
   - **Тариф:** минимальный (1 CPU, 1 GB RAM, 10 GB SSD — достаточно для статики)
   - **Локация:** Москва или Санкт-Петербург (для скорости в РФ)
3. Нажми **«Создать»**
4. Через 2–3 минуты сервер будет готов. Скопируй IP-адрес и root-пароль из панели.

### Шаг 2. Подключение к серверу

```bash
# На Mac / Linux — открой терминал
ssh root@IP_АДРЕС_СЕРВЕРА

# На Windows — используй PuTTY или встроенный OpenSSH:
# ssh root@IP_АДРЕС_СЕРВЕРА
```

При первом подключении введи `yes`, затем пароль из панели Timeweb.

### Шаг 3. Установка необходимого ПО

```bash
# Обновление системы
apt update && apt upgrade -y

# Установка Node.js 20 (LTS)
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# Проверка
node -v  # должен выдать v20.x.x
npm -v   # должен выдать 10.x.x

# Установка Nginx
apt install -y nginx

# Установка Git
apt install -y git
```

### Шаг 4. Клонирование репозитория

```bash
# Создаём папку для проекта
mkdir -p /var/www/aikod
cd /var/www/aikod

# Клонируем репозиторий (замени URL на свой)
git clone https://github.com/ТВОЙ_ЮЗЕРНЕЙМ/ТВОЙ_РЕПОЗИТОРИЙ.git .

# Если репозиторий приватный — используй SSH:
# git clone git@github.com:ТВОЙ_ЮЗЕРНЕЙМ/ТВОЙ_РЕПОЗИТОРИЙ.git .
```

### Шаг 5. Сборка проекта

```bash
cd /var/www/aikod

# Установка зависимостей
npm install

# Сборка для production
npm run build

# Проверка — должна появиться папка dist/
ls -la dist/
```

### Шаг 6. Настройка Nginx

Файл конфигурации уже лежит в репозитории: `nginx.conf`

```bash
# Копируем конфигурацию
cp /var/www/aikod/nginx.conf /etc/nginx/sites-available/aikod

# Создаём симлинк
ln -s /etc/nginx/sites-available/aikod /etc/nginx/sites-enabled/

# Удаляем дефолтный конфиг (чтобы не мешал)
rm /etc/nginx/sites-enabled/default

# Проверяем синтаксис
nginx -t

# Перезапускаем Nginx
systemctl restart nginx
systemctl enable nginx  # автозапуск при ребуте
```

### Шаг 7. Настройка фаервола

```bash
# Установка UFW (если не установлен)
apt install -y ufw

# Разрешаем SSH, HTTP, HTTPS
ufw allow OpenSSH
ufw allow 'Nginx Full'

# Включаем фаервол
ufw enable

# Проверка статуса
ufw status
```

### Шаг 8. SSL-сертификат (HTTPS)

```bash
# Установка Certbot
apt install -y certbot python3-certbot-nginx

# Получение сертификата Let's Encrypt (замени домен)
certbot --nginx -d aikod.sergeichernenko.ru

# Следуй инструкциям:
# 1. Введи email
# 2. Согласись с условиями
# 3. Выбери перенаправление HTTP → HTTPS (рекомендуется 2)

# Автоматическое обновление сертификата (запускается автоматически, проверим)
certbot renew --dry-run
```

### Шаг 9. Привязка домена

1. Перейди в панель управления доменом `sergeichernenko.ru`
2. Добавь A-запись:
   - **Имя:** `aikod`
   - **Тип:** `A`
   - **Значение:** `IP_АДРЕС_ТВОЕГО_VDS`
3. Сохрани и жди 5–30 минут

### Шаг 10. Проверка

Открой в браузере: `https://aikod.sergeichernenko.ru`

Сайт должен открываться по HTTPS.

### Шаг 11. Автодеплой через GitHub Actions (опционально)

В репозитории уже создан файл `.github/workflows/deploy.yml`.  
Чтобы он работал:

1. В репозитории на GitHub перейди в **Settings → Secrets and variables → Actions**
2. Добавь секреты:
   - `VPS_HOST` — IP-адрес твоего VDS
   - `VPS_USER` — `root` (или созданный пользователь)
   - `VPS_SSH_KEY` — приватный SSH-ключ (содержимое файла `~/.ssh/id_rsa` с сервера)
3. Теперь при каждом push в `main` GitHub Actions автоматически:
   - Подключится к серверу по SSH
   - Сделает `git pull`
   - Выполнит `npm install && npm run build`
   - Перезапустит Nginx

## Проблемы и решения

### «Сайт открывается, но при обновлении страницы — 404»

Это проблема SPA-роутинга. Nginx конфиг уже исправлен (`try_files`), но если используешь Apps — убедись, что в настройках включён **«SPA fallback»** или аналогичная опция.

### «Шрифты/картинки не загружаются»

Проверь в DevTools (F12 → Network) — возможно, пути начинаются с `/` а не `./`. В Vite всё корректно, но если картинки лежат в `public/` — они копируются как есть.

### «SSL не работает»

Для VDS: убедись, что `certbot` успешно прошёл и Nginx перезапущен.  
Для Apps: SSL выдаётся автоматически, подожди 5–15 минут.

## Рекомендуемый выбор

| Критерий | Timeweb Apps | VDS + Nginx |
|----------|-------------|-------------|
| **Сложность** | Очень низкая | Средняя |
| **Цена** | ~200–400 ₽/мес | ~150–300 ₽/мес |
| **Время настройки** | 10 минут | 30–40 минут |
| **Автодеплой** | Встроен | Через GitHub Actions |
| **SSL** | Авто | Ручная настройка (1 команда) |
| **Масштабирование** | Авто | Вручную |

**Для этого проекта рекомендую Timeweb Cloud Apps** — быстрее, проще, надёжнее. VDS подойдёт, если планируешь размещать несколько сайтов или хочешь полный контроль.
